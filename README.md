# Информационная система мебельного магазина

Курсовой проект на ASP.NET Core MVC с базой данных SQLite и Entity Framework Core.

## Возможности

- Каталог мебели с поиском по названию, артикулу и производителю.
- CRUD-разделы для администратора: каталог, категории, производители, клиенты, заказы.
- Простая авторизация и регистрация без ASP.NET Identity.
- Роли `Admin` и `User` с ограничением доступа.
- Корзина пользователя и оформление заказа.
- История заказов со статусами `В обработке` и `Выполнен`.
- Загрузка фото мебели с компьютера.
- Автоматическое применение миграций и заполнение тестовыми данными при первом запуске.

## Требования

- .NET SDK 8.0 или новее
- Docker и Docker Compose, если нужен запуск в контейнере

## Локальный запуск

```bash
dotnet restore
dotnet ef database update
dotnet run
```

После запуска откройте адрес, который покажет команда `dotnet run`.

## Запуск через Docker

Собрать и запустить контейнер:

```bash
docker compose up -d --build
```

Открыть сайт:

```text
http://localhost:5000
```

Остановка:

```bash
docker compose down
```

Просмотр логов:

```bash
docker compose logs -f
```

## База данных

SQLite хранится в файле:

```text
data/furniture_store.db
```

Приложение само создает папку `data`, применяет миграции при запуске и заполняет базу тестовыми данными.

В Docker папка `./data` проброшена в контейнер как `/app/data`, поэтому база сохраняется между перезапусками контейнера.

Загруженные фотографии мебели хранятся в:

```text
wwwroot/uploads
```

В Docker эта папка также проброшена как volume.

## Миграции

Если инструмент EF Core не установлен:

```bash
dotnet tool install --global dotnet-ef
```

Применить миграции вручную:

```bash
dotnet ef database update
```

Создать новую миграцию после изменения моделей:

```bash
dotnet ef migrations add AddNextChanges
dotnet ef database update
```
